# AutoRia Clone API (Final Project)

## Запуск проєкту
```
npm install
```

## Через Docker (Рекомендовано)
Система повністю контейнеризована. Це найшвидший спосіб запуску, оскільки всі залежності та білд-процеси вже налаштовані всередині образу.

Переконайтеся, що Docker Desktop запущений.

Виконайте команду в корені проєкту:

```
docker-compose up --build
```

API буде доступне за адресою: **http://localhost:3333**

## Локальний запуск (без Docker)

Якщо ви хочете запустити проєкт напряму на вашій машині Встановіть залежності:

```
npm install
```

Запустіть проєкт у режимі розробки:

```
npm run dev
```

## 📂 Тестування через Postman (Postman Files)

У корені проєкту підготовлено **два файли** для швидкого та повного тестування всієї бізнес-логіки:

1.  **`express-project.postman_collection.json`** — основна колекція з усіма запитами (Auth, Cars, Users, Admin), розбитими за логічними папками.
2.  **`express-p.postman_environment.json`** — файл оточення, що містить глобальні змінні для проєкту.

### 🛠 Як налаштувати для перевірки:
1.  **Імпортуйте обидва файли** у ваш Postman (кнопка *Import*).
2.  У правому верхньому куті Postman оберіть оточення (Environment).
3.  **Змінні**: У файлі оточення вже прописана `base_url = http://localhost:3333`.
4.  **Автоматизація**: Колекція налаштована так, що після успішного логіну (`Login`) отриманий **JWT Access Token** автоматично зберігається у змінну оточення і підставляється в усі наступні запити. Вам не потрібно копіювати токени вручну.
5. На всякий випадок код для скріпта в логінації:
```
const response = pm.response.json();
if (response.data && response.data.tokens) {
    pm.environment.set("accessToken", response.data.tokens.accessToken);
    pm.environment.set("refreshToken", response.data.tokens.refreshToken);
}
```

## ⚙️ Основні можливості та Бізнес-логіка

Проєкт реалізований з урахуванням високих вимог до масштабованості та гнучкості архітектури.

### 🏗️ Архітектура та Масштабованість
- **Система пермішінів**: Замість жорстких ролей використано систему дозволів. Це дозволяє в майбутньому легко впроваджувати функціонал **"Автосалонів"** (додавання механіків, сейлз-менеджерів та адмінів конкретних салонів) без переписування ядра системи.
- **N-tier Architecture**: Чіткий розподіл на контролери, сервіси, репозиторії та мідлвари.

### 🛡️ Розумна Модерація (Anti-Abuse)
- **Автоматичний фільтр**: Кожне оголошення при створенні або редагуванні проходить через фільтр нецензурної лексики.
- **Ліміт спроб**: Продавець має лише **3 шанси** відредагувати текст, щоб пройти модерацію.
- **Система статусів**: При виявленні матірних слів оголошення отримує статус "Pending/Inactive".
- **Сповіщення**: Якщо за 3 спроби модерація не пройдена, система автоматично надсилає імейл-звіт менеджеру (через **Ethereal**) для прийняття рішення про бан або видалення.

### 💰 Монетизація та типи акаунтів
- **Account Types**: Реалізовано два типи — `Base` (безкоштовний) та `Premium` (платний).
- **Ліміти**:
    - Користувач з `Base` акаунтом може мати лише **1 активне оголошення**.
    - `Premium` користувачі мають **безліміт** на кількість оголошень.

### 📈 Аналітика (тільки для Premium)
Преміум-користувачі отримують доступ до детальної статистики, щоб краще розуміти ринок:
- **Перегляди**: Загальна кількість та статистика за періодами (день, тиждень, місяць).
- **Market Price**: Середня ціна на аналогічне авто по конкретному **регіону** (області).
- **Ukraine Price**: Середня ціна на таку модель по всій території України.

### 💱 Робота з валютами та API ПриватБанку
- **Динамічні курси**: Система автоматично синхронізується з API ПриватБанку раз на добу для актуалізації курсів USD, EUR та UAH.
- **Гнучкість цін**: Продавець вказує ціну в одній валюті, а система автоматично вираховує та зберігає еквіваленти в інших.
- **Прозорість**: В оголошенні фіксується курс, за яким проводився розрахунок на момент публікації.

### 🚗 Довідники авто (Brands & Models)
- **Database Driven**: Усі доступні марки та моделі зберігаються в базі даних у колекції `brands`.
- **Зв'язок**: Реалізовано чітку ієрархію `Brand -> Model` (наприклад, BMW -> X5, Daewoo -> Lanos).
- **Розширення**: Якщо користувач не знаходить потрібної марки у випадаючому списку, передбачено механізм повідомлення адміністрації для оновлення довідника.

### 🏗️ Архітектура та масштабованість (Ролі)
- **Role-Based Access Control (RBAC)**: На даному етапі доступ розмежовано через систему ролей (`Admin`, `Manager`, `Seller`, `Buyer`).
- **Перспектива розвитку**: Моделі користувачів спроєктовані таким чином, щоб у майбутньому легко перейти на розширену систему **пермішінів**. Це дозволить масштабувати платформу для великих автосалонів, де потрібен гнучкий розподіл прав між механіками, сейлз-менеджерами та директорами філій.
